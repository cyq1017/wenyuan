# 文渊使用指南

> 文渊的全部功能和使用方法。建议先完成[[快速上手教程]]再来查阅本指南。

---

## 每日工作流

推荐的日常使用节奏：

1. **早上** — `/start-my-day`：回顾昨日、规划今日、记录新想法
2. **工作中** — 根据计划使用具体命令：
   - 读文献用 `/read`
   - 写论文用 `/write`
   - 有新想法随时记入 `00_收件箱/`
3. **晚上** — 回到今日日记，补充"当日总结"部分

---

## 核心命令速查

### 每日规划

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/start-my-day` | 晨间学术规划 | 每天开始工作时 |

### 研究阶段

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/brainstorm` | 头脑风暴 | 有模糊想法需要梳理 |
| `/explore <概念>` | 概念深度梳理 | 遇到不熟悉的学术概念 |
| `/research <主题>` | 系统性调研 | 需要全面了解一个主题 |
| `/read` | 引导式精读 | 精读一篇论文或著作 |

### 写作阶段

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/kickoff` | 想法变项目 | 确定了研究方向，正式立项 |
| `/write` | 写作辅助 | 润色段落、检查逻辑、改学术用语 |

### 管理维护

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/organize` | 整理笔记 | 有零散笔记需要归类 |
| `/ask <问题>` | 快速问答 | 有学术问题想快速了解 |
| `/academic-feed` | 学术动态 | 查看教职/论文/会议信息 |
| `/archive` | 归档 | 清理已完成的项目 |

## 目录结构详解

### `00_收件箱/` — 灵感捕捉站
- 想到什么就往这里扔，不需要整理
- 每一条一个 `.md` 文件
- 用 `/kickoff`（转项目）、`/read`（转文献）、`/explore`（转概念）处理后自动标记 `processed`

### `10_日记/` — 学术日志
- 每天一个文件：`YYYY-MM-DD.md`
- 由 `/start-my-day` 自动创建
- 记录今日计划、阅读思考、写作进展、灵感和总结

### `20_项目/` — 学术项目
- 一个项目一个文件（或文件夹）
- 适用于：论文、课题申报、研究计划、读书报告等
- 使用 C.A.P. 结构：Context（背景）、Actions（行动）、Progress（进展）
- 通过 frontmatter 中的 `status` 字段管理：`active` / `on-hold` / `done`

### `30_文献/` — 文献精读笔记
- 由 `/read` 生成
- 命名规则：`作者姓_年份_简短标题.md`
- 包含核心论点、研究方法、关键发现、与自己研究的关系

### `40_概念/` — 学术概念卡片
- 由 `/explore` 生成
- 一个概念一张卡片（原子化）
- 包含定义、学术脉络、代表人物、相关概念

### `50_资源/` — 外部资源
- `期刊动态/` — 期刊征稿、会议信息
- `学术工具/` — 研究方法、写作工具等

### `90_计划/` — 写作计划
- 阶段性的研究计划和目标
- 完成后由 `/archive` 归档

### `99_系统/` — 系统配置
- `模板/` — Obsidian 笔记模板
- `提示词/` — AI 角色设定
- `数据库/` — Obsidian Bases 视图
- `偏好设置.md` — 你的研究领域和关注偏好
- `归档/` — 已完成项目的存放处

## 项目标准格式（C.A.P.）

每个项目遵循 Context-Actions-Progress 结构：

```markdown
## Context（背景）
### 研究问题
### 研究意义
### 预期成果

## Actions（行动）
### 第一阶段：选题与文献综述
- [ ] 确定研究问题
- [ ] 文献检索与筛选

### 第二阶段：研究设计
- [ ] 确定研究方法

### 第三阶段：撰写与修改
- [ ] 初稿撰写

## Progress（进展）
### YYYY-MM-DD
- 进展：...
- 下一步：...
```

## 收件箱工作流

```
随时记录 ──────────→ 00_收件箱/（手动创建）
start-my-day Q2 ──→ 00_收件箱/（AI 自动创建）
                        │
                ┌───────┼────────┐
                ↓       ↓        ↓
            /kickoff  /read   /explore
            想法→项目  →文献    →概念
                │       │        │
                ↓       ↓        ↓
            20_项目/  30_文献/  40_概念/

处理后标记 status: processed
定期用 /archive 归档已处理条目
```

## 偏好设置

编辑 `99_系统/偏好设置.md` 来定制你的文渊：

- **研究领域**：填写你的主领域和关注方向
- **学术动态**：控制 `/academic-feed` 推送哪些类型的信息
- **引用格式**：默认 GB/T 7714，可改为 APA、MLA、Chicago 等

## 进阶

### Obsidian 插件推荐（按需选装）

| 插件 | 用途 | 推荐度 |
|------|------|--------|
| Calendar | 日历视图查看日记 | 建议安装 |
| Dataview | 将笔记库变成可查询的数据库 | 进阶用户 |
| Templater | 更灵活的模板自动化 | 进阶用户 |
| ZotLit | Zotero 文献管理集成 | 使用 Zotero 的用户 |
| PDF++ | Obsidian 内阅读和批注 PDF | 常读 PDF 的用户 |
| Obsidian Git | 自动备份到 GitHub | 建议安装 |

### 多端同步

| 方案 | 适用场景 | 难度 |
|------|----------|------|
| iCloud | 纯苹果生态（Mac + iPhone） | 零配置 |
| Obsidian Sync（付费） | 跨平台、最省心 | 零配置 |
| Remotely Save 插件 | 有 WebDAV/NAS | 中等 |

### 个性化定制

- **修改文件夹**：根据你的研究习惯调整文件夹名称和结构
- **自建 skill**：在 `.claude/skills/` 下创建新的 `SKILL.md` 定义你自己的工作流
- **修改模板**：在 `99_系统/模板/` 下修改或新增模板
