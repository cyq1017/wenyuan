# 文渊使用指南

> 文渊的全部功能和使用方法。建议先完成[[快速上手教程]](快速上手教程.md)再来查阅本指南。

---

## 每日工作流

推荐的日常使用节奏：

1. **早上** — `/start-my-day`：回顾昨日、规划今日、推送学术动态（教职/会议/新论文）、生成当天日记
2. **工作中** — 根据计划使用具体命令：
   - 读文献用 `/read`，写论文用 `/write`
   - 研究新课题用 `/research`，梳理概念用 `/explore`
   - 想法不成熟用 `/brainstorm`，快速查询用 `/ask`
   - 有新想法随时记入 `00_收件箱/`
   - 课堂笔记、读书摘录等用 `/parse` 拆解为结构化笔记
3. **晚上** — `/end-my-day`：复盘今日、更新项目进展、归档对话、规划明日

## 核心命令速查

### 每日规划

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/start-my-day` | 晨间学术规划 | 每天开始工作时 |
| `/end-my-day` | 每日复盘总结 | 每天结束工作时 |

### 研究阶段

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/brainstorm` | 头脑风暴 | 有模糊想法需要梳理 |
| `/explore <概念>` | 概念深度梳理 | 遇到不熟悉的学术概念 |
| `/research <主题>` | 系统性调研 | 需要全面了解一个主题 |
| `/read` | 引导式精读 | 精读一篇论文或著作 |

### 写作阶段

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/kickoff` | 想法变项目 | 确定了研究方向，正式立项（AI 先出计划，你确认后再创建） |
| `/write` | 写作辅助 | 润色段落、检查逻辑、改学术用语 |

### 管理维护

| 命令 | 功能 | 何时用 |
|------|------|--------|
| `/organize` | 整理笔记 | 有零散笔记需要归类 |
| `/parse` | 知识解析 | 把课堂笔记、读书摘录等拆解为笔记 + 概念卡片 |
| `/ask <问题>` | 快速问答 | 写作中随手查一个概念 |
| `/academic-feed` | 学术动态 | 已集成到 start-my-day，也可单独深度搜索 |
| `/archive` | 归档 | 清理已完成的项目 |

## 目录结构详解

### `00_收件箱/` — 灵感捕捉站
- 想到什么就往这里扔，不需要整理
- 每一条一个 `.md` 文件
- 用 `/kickoff`（转项目）、`/read`（转文献）、`/explore`（转概念）处理后自动标记 `processed`

### `10_日记/` — 学术日志
- 每天一个文件：`YYYY-MM-DD.md`
- 由 `/start-my-day` 自动创建
- 记录四个方面：今日计划、阅读与思考、项目进展、灵感与想法

### `20_项目/` — 学术项目
- 一个项目一个文件（或文件夹）
- 适用于：论文、课题申报、研究计划、读书报告等
- 使用 C.A.P. 结构：Context（背景）、Actions（行动）、Progress（进展）
- 通过 frontmatter 中的 `status` 字段管理：`active` / `on-hold` / `done`

### `30_文献/` — 文献精读笔记
- 由 `/read` 生成
- 命名规则：`作者姓_年份_简短标题.md`
- 包含核心论点、研究方法、关键发现、与自己研究的关系

### `40_概念/` — 学术概念卡片
- 由 `/explore` 生成
- 一个概念一张卡片（原子化）
- 包含定义、学术脉络、代表人物、相关概念

### `50_资源/` — 外部资源
- `学术动态/` — 每日推送的学术信息 digest（由 start-my-day 自动生成）
- `期刊动态/` — 期刊征稿、会议信息
- `学术工具/` — 研究方法、写作工具等

### `90_计划/` — 写作计划
- 阶段性的研究计划和目标
- 完成后由 `/archive` 归档

### `99_系统/` — 系统配置
- `模板/` — Obsidian 笔记模板
- `提示词/` — AI 角色设定
- `数据库/` — Obsidian Bases 数据库视图（用表格形式浏览和管理笔记）
    - `文献管理.base` — 按状态、领域筛选文献笔记
    - `概念索引.base` — 按分类浏览概念卡片
    - `论文进度.base` — 按截止日期跟踪项目进展
    - `项目看板.base` — 按状态/优先级查看所有项目
    - 打开方式：在 Obsidian 中直接点击 `.base` 文件即可看到表格视图
- `偏好设置.md` — 你的研究领域和关注偏好
- `归档/` — 已完成项目的存放处

## 项目标准格式（C.A.P.）

每个项目遵循 Context-Actions-Progress 结构：

```markdown
## Context（背景）
### 研究问题
### 研究意义
### 预期成果

## Actions（行动）
### 第一阶段：选题与文献综述
- [ ] 确定研究问题
- [ ] 文献检索与筛选

### 第二阶段：研究设计
- [ ] 确定研究方法

### 第三阶段：撰写与修改
- [ ] 初稿撰写

## Progress（进展）
### YYYY-MM-DD
- 进展：...
- 下一步：...
```

```
                ┌───────┼────────┐

            /kickoff  /read   /explore
            想法→项目  →文献    →概念
                │       │        │
                ↓       ↓        ↓
            20_项目/  30_文献/  40_概念/

处理后标记 status: processed
定期用 /archive 归档已处理条目
```

## 偏好设置

编辑 `99_系统/偏好设置.md` 来定制你的文渊：

- **研究领域**：填写你的主领域和关注方向
- **信息订阅**：设定你关注的学术动态类型（教职/会议/论文/基金）和非学术兴趣
- **引用格式**：默认 GB/T 7714，可改为 APA、MLA、Chicago 等

> 偏好设置是一个普通 md 文件，随时可以修改。也可以直接跟 Claude Code 说“我现在对 XX 也感兴趣了”，它会帮你更新。

## 进阶

### Obsidian 插件推荐（按需选装）

| 插件 | 用途 | 推荐度 |
|------|------|--------|
| Calendar | 日历视图查看日记 | 建议安装 |
| Dataview | 将笔记库变成可查询的数据库 | 进阶用户 |
| Templater | 更灵活的模板自动化 | 进阶用户 |
| ZotLit | Zotero 文献管理集成 | 使用 Zotero 的用户 |
| PDF++ | Obsidian 内阅读和批注 PDF | 常读 PDF 的用户 |
| Obsidian Git | 自动备份到 GitHub | 建议安装 |

### 多端同步

| 方案 | 适用场景 | 难度 |
|------|----------|------|
| iCloud | 纯苹果生态（Mac + iPhone） | 零配置 |
| Obsidian Sync（付费） | 跨平台、最省心 | 零配置 |
| Remotely Save 插件 | 有 WebDAV/NAS | 中等 |

**iCloud 同步：** 将文渊 vault 放到 `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/` 目录下，iPhone 上的 Obsidian App 会自动同步。

### 个性化定制

文渊的每一个部分都可以根据你的习惯调整。

> 建议：把你想要的改动先告诉 Claude Code，一起商量好再执行，让它帮你管理和更新。这样做更靠谱且省事，也能帮你养成使用 Claude Code 的习惯。

#### 修改文件夹结构

文渊的文件夹是建议性的，你可以：
- 改名：比如 `20_项目/` → `20_论文/`（如果你只写论文）
- 新增：比如 `60_教学/` 放教案和课件
- 删除：不用的文件夹可以删掉

**注意：** 改文件夹后，需要同步更新 `CLAUDE.md` 和 `.claude/settings.json` 中的路径引用，否则 AI 会找不到对应的目录。**建议直接让 Claude Code 帮你完成这些更新**——告诉它你改了什么，它会自动同步所有引用，避免遗漏。

#### 修改模板

模板位于 `99_系统/模板/`，你可以：

1. **调整现有模板**：打开模板文件，增删你需要的字段
2. **创建新模板**：复制一个现有模板，修改后另存为新名称
3. **在 Obsidian 中设置**：如果安装了 Templater 插件，可以实现快捷键插入模板

示例：如果你的导师要求项目计划中必须包含"伦理审查"部分，直接在 `项目模板.md` 中加上即可。

#### 自建 Skill（工作流）

如果你有一个经常重复的研究任务（比如每周做一次文献追踪），可以创建自己的 skill：

1. 在 `.claude/skills/` 下新建一个文件夹，如 `weekly-review/`
2. 在其中创建 `SKILL.md`，按以下格式写：

```markdown
---
name: weekly-review
description: 每周文献追踪 — 回顾本周阅读，追踪关注领域新发表
---

# 每周文献回顾

你的具体指令写在这里...

## 步骤
1. 扫描本周日记中提到的文献
2. 搜索关注领域的新论文
3. 生成周报保存到 50_资源/
```

3. 保存后，在 Claude Code 中输入 `/weekly-review` 即可触发
4. **不知道怎么写？** 直接跟 Claude Code 聊你的想法："我每周都要做一次文献追踪，帮我写成一个 skill"——它会帮你生成完整的 SKILL.md

#### 修改偏好设置

编辑 `99_系统/偏好设置.md`：
- **研究领域**：AI 会根据你的领域定制搜索和推荐
- **学术动态开关**：关闭不感兴趣的推送类型
- **引用格式**：改成你学校/期刊要求的格式

#### AI 角色切换

文渊内置了多种 AI 角色，适用于不同的研究阶段。角色设定文件位于 `99_系统/提示词/`：

| 角色 | 风格 | 适用场景 |
|------|------|----------|
| **学术伙伴**（默认） | 温暖鼓励，苏格拉底式提问 | 日常对话、`/start-my-day`、`/brainstorm` |
| **论文教练** | 严格直接，指出问题 | `/write` 终稿审查、论证逻辑检查 |
| **方法论顾问** | 结构化分析，方案对比 | `/research` 设计研究方案、选方法 |
| **批判性阅读者** | 质疑假设，找逻辑漏洞 | `/read` 精读文献、文献综述评估 |
| **答辩模拟** | 模拟考官，压力测试 | 毕业答辩/教职面试准备 |

**切换方式：** 直接在对话中说即可——

- "用论文教练模式帮我看这段"
- "切换到批判性阅读者，帮我挑这篇论文的毛病"
- "模拟一下答辩，我明天要开题"

你也可以编辑 `99_系统/提示词/` 中的文件来调整角色行为，或者在 `CLAUDE.md` 的"沟通风格"部分做全局调整。比如加上：
```
- 回答尽量简短，不超过 3 段
- 不要使用 emoji
- 优先使用英文学术术语
```
