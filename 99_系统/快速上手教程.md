# 快速上手教程

> 跟着走一遍，体验文渊的核心工作流：**想法 → 计划 → 项目 → 执行 → 积累**。大约 20 分钟。

---

## 准备

1. 在 Obsidian 中打开文渊文件夹
2. 安装必要插件（如果还没装）：`设置` → `第三方插件` → `浏览` → 搜索并安装 **Terminal**。详见 [[Obsidian新手指南]](Obsidian新手指南.md)
3. 打开 Terminal 插件的终端面板：`Cmd/Ctrl + P` → 搜索 `Terminal: Open terminal`
4. 在终端中运行 `claude`

---

## Step 1：先写下你的想法

全新的文渊还没有任何内容。在开始之前，先在 `00_收件箱/` 中写下你的第一个想法。

例如，新建一个文件 `00_收件箱/distant-reading.md`，写下：

> 我上周读了一篇关于"数字人文"的文章，里面提到了 distant reading 的方法，想搞清楚这跟传统的 close reading 有什么区别，能不能用在我的论文里。

这就是一个典型的收件箱条目——一个不成熟但值得追踪的想法。

---

## Step 2：开始新的一天

现在收件箱里有内容了，运行：

```
/start-my-day
```

AI 会：
- 读取收件箱中的想法，纳入今日计划
- 问你今天的研究重点
- **推送学术动态**（首次使用时会问你关注哪些领域，并写入偏好设置）
- 生成今天的日记

接下来，我们就围绕刚才写下的 distant reading 这个想法，体验完整的工作流。

---

## Step 3：头脑风暴

这个想法还比较模糊，用头脑风暴梳理一下：

```
/brainstorm
```

AI 会用开放性问题引导你思考：
- "你对这个问题的哪个方面最感兴趣？"
- "想从什么角度切入？"
- "已有研究做过类似探索吗？"

聊完后，AI 整理出讨论要点和可能的研究方向。

---

## Step 4：探索核心概念

头脑风暴中提到了"东方主义"和"文化霸权"，深入了解一下：

```
/explore 文化霸权
```

AI 生成一张概念卡片 → 保存到 `40_概念/文化霸权.md`，包含定义、学术脉络、代表人物、相关概念的双向链接。

---

## Step 5：系统调研

想对这个选题做一次完整的文献调研：

```
/research 中国文学海外传播与东方主义
```

AI 先生成调研计划让你确认，然后系统性执行：
- 生成研究综述 → `30_研究/`
- 创建/更新概念卡片 → `40_概念/`
- 推荐关键文献

---

## Step 6：精读一篇关键文献

调研中发现赛义德的《东方主义》是绕不开的文献：

```
/read
```

AI 会先问你要精读什么。你可以：
- **说书名**："帮我精读赛义德的《东方主义》"
- **粘贴摘要**：贴入论文的标题、作者、摘要等信息
- **指向已有笔记**："精读收件箱里那篇关于 distant reading 的笔记"

然后 AI 用苏格拉底式提问引导你精读："核心论点是什么？""方法论是什么？""对你的研究有什么启发？"

对话后生成文献笔记 `赛义德_1978_东方主义.md` → `30_研究/`

---

## Step 7：把想法变成项目

经过调研和精读，想法已经足够清晰，正式立项：

```
/kickoff
```

AI 引导你填写研究问题、意义、阶段计划，生成项目文件 → `20_项目/`。

---

## Step 8：快速问一个问题

写项目计划时，想快速了解"再现理论"：

```
/ask 什么是再现理论？
```

AI 直接给简洁回答，不生成大量文档。这就是 `/ask` 的用法——写作中随手查概念，轻量快速。

---

## Step 9：写作辅助

写了一段研究背景，想让 AI 优化：

```
/write
```

AI 改善学术表达、检查逻辑、建议用词——保留你的核心观点。

---

## Step 10：结束一天

工作结束时：

```
/end-my-day
```

AI 会：
- 汇总你今天的成果（新建了哪些笔记、推进了哪些项目）
- 更新项目的 Progress 记录
- 归档今日对话摘要
- 生成明日重点

> 💡 项目完成后，随时可以用 `/archive` 清理归档，保持文渊整洁。

---

## Step 11：定制文渊（进阶）

用熟了之后，文渊可以变成完全属于你的系统。

### 自建 Skill

你发现经常需要把课堂纸质笔记 OCR 归档——这种重复性任务可以做成 skill：

```markdown
---
name: ocr-notes
description: 课堂笔记 OCR — 拍照识别手写笔记并归档到 vault
---

# 课堂笔记 OCR 归档

## 步骤
1. 用户提供笔记照片或文字
2. 识别/整理内容
3. 分析内容类型（文献信息、概念、研究想法）
4. 按类型创建对应笔记到正确位置
5. 建立 wikilinks 关联
```

保存到 `.claude/skills/ocr-notes/SKILL.md`，输入 `/ocr-notes` 即可触发。

**不知道怎么写？** 直接跟 Claude Code 聊："我经常需要把课堂笔记拍照整理成文献笔记，帮我写一个 skill"——它会帮你完成。

### 定制工作流

你也可以修改现有 skill 的行为。例如，让 `/start-my-day` 额外推送你关注的行业动态，或让 `/end-my-day` 自动生成周报。直接告诉 Claude Code 你的想法，它会帮你调整 skill 文件。

### 多仓库管理

文渊可以作为你的核心知识库，同时管理其他 Obsidian vault。例如：

- **文渊**（学术主库）：论文、文献、项目
- **读书笔记**：非学术阅读
- **个人生活**：穿搭方案、美容养生、旅行计划…

通过 Obsidian CLI 或 Claude Code，你可以在文渊中引用和搜索其他 vault 的内容。

### 知识解析

有一堆课堂笔记或读书摘录？用 `/parse` 让 AI 帮你拆解：

输入 `/parse`，然后粘贴你的笔记内容。AI 会把它拆解为：
- 一份结构化的主笔记（放入 30_研究/）
- 若干概念卡片（放入 40_概念/，自动建立 wikilinks）
- 待跟进的想法（放入收件箱）

AI 会先展示解析方案让你确认，再执行创建。

> 更多定制方法见 [文渊使用指南](文渊使用指南.md#个性化定制)。

## 回顾

你体验了文渊的完整闭环：

```
想法  ──→  /brainstorm  ──→  清晰方向
  ↓
计划  ──→  /explore + /research  ──→  理论基础
  ↓
立项  ──→  /kickoff  ──→  结构化项目
  ↓
执行  ──→  /read + /write  ──→  文献 + 文本
  ↓
积累  ──→  /end-my-day  ──→  记录 + 归档
  ↓
定制  ──→  自建 skills  ──→  个人工作流
```

随着使用，你的文渊会积累越来越多的文献笔记、概念卡片和项目记录，通过双向链接编织成属于你的学术知识网络。
