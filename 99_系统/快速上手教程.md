# 快速上手教程

> 通过一个完整场景，带你认识文渊的核心功能。大约需要 15 分钟。

---

## 准备

1. 确保你已经安装了 Obsidian 和 Claude Code（参见 [[README]]）
2. 已在 Obsidian 中打开了文渊文件夹
3. 打开终端，`cd` 到文渊所在的文件夹
4. 运行 `claude` 启动 AI 助手

---

## 场景：把一条灵感变成研究项目

假设你在课堂上想到一个有趣的研究方向，我们来看文渊如何帮你从"一闪念"走到"可执行的项目"。

---

### Step 1：开始新的一天

在 Claude Code 中输入：

```
/start-my-day
```

AI 会：
- 检查你昨天的日记和未完成事项
- 问你今天想做什么
- 问你有没有新想法
- 生成今天的学术日志

第一次使用时，由于还没有历史数据，AI 会帮你创建第一篇日记。

---

### Step 2：记录灵感到收件箱

假设你想到："课堂笔记能不能用 AI 自动归类到知识体系里？"

你可以直接在 Obsidian 中新建一个文件：

1. 打开 `00_收件箱/` 文件夹
2. 新建文件，写下你的想法
3. 或者直接在 `/start-my-day` 的对话中说出来，AI 会自动帮你创建

---

### Step 3：头脑风暴

这个想法还比较模糊。在 Claude Code 中输入：

```
/brainstorm
```

AI 会用发散性提问帮你梳理：
- "你对这个问题的哪个方面最感兴趣？"
- "你想从技术角度还是教育学角度研究？"
- "已有研究做过类似的探索吗？"

聊完后，AI 会整理出讨论要点和可能的研究方向。

---

### Step 4：探索核心概念

在头脑风暴中你提到了"知识管理"这个概念。深入了解它：

```
/explore 知识管理
```

AI 会生成一张概念卡片，包含：
- 定义（不同学者的理解）
- 学术脉络（从哪里来、怎么发展的）
- 代表人物和著作
- 相关概念（学习理论、信息素养等）

卡片会自动保存到 `40_概念/知识管理.md`。

---

### Step 5：调研一个主题

想更系统地了解这个领域：

```
/research AI辅助学术知识管理
```

AI 会：
1. 先生成一份调研计划（涉及哪些方向、框架）
2. 你确认后，AI 开始系统调研
3. 最终生成一份研究综述，保存到 `30_文献/`
4. 同时更新相关的概念卡片

---

### Step 6：精读一篇关键文献

在调研中发现了一篇重要的论文，想深入精读：

```
/read
```

AI 会用苏格拉底式提问引导你思考：
- "你觉得这篇文献的核心论点是什么？"
- "作者的研究方法是什么？"
- "这篇文献对你的研究有什么启发？"

对话结束后，AI 生成结构化的文献笔记，保存到 `30_文献/`。

---

### Step 7：把想法变成项目

经过调研和阅读，你的想法已经足够清晰：

```
/kickoff
```

AI 引导你填写：
- 研究问题
- 研究意义
- 初步计划和阶段目标

最终生成一个项目文件，保存到 `20_项目/`，使用 C.A.P.（Context-Actions-Progress）结构。

---

### Step 8：润色一段论述

开始写项目的开题报告，写了一段想让 AI 帮你优化：

```
/write
```

粘贴你的草稿，AI 会：
- 改善语言流畅度
- 检查论证逻辑
- 建议更好的学术表达
- 解释每处修改的理由

---

### Step 9：回到日记总结

一天结束，回到今天的日记 `10_日记/YYYY-MM-DD.md`，在"当日总结"部分记录：
- 完成了什么
- 没完成什么
- 明天的重点

---

## 恭喜！

你已经体验了文渊的核心功能：

```
灵感（收件箱）
    ↓ /brainstorm
清晰想法
    ↓ /explore + /research
理论基础
    ↓ /read
文献支撑
    ↓ /kickoff
结构化项目
    ↓ /write
学术文本
    ↓ /archive
归档整理
```

下一步：
- 查看 [[文渊使用指南]] 了解全部功能
- 编辑 [[偏好设置]] 定制你的研究领域
- 开始你自己的研究之旅！
