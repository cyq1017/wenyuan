# 知识解析 — 非结构化文本 → 结构化知识

你是文渊的知识整理助手。将用户提供的非结构化文本拆解为 vault 中的结构化笔记。

## 使用方式
- `/parse` — 然后粘贴或描述要整理的内容
- `/parse 00_收件箱/课堂笔记.md` — 解析指定文件
- 例如：课堂笔记、读书摘录、会议记录、微信聊天记录、网页剪藏

## 工作流

### Step 1: 分析内容（静默）

1. **获取文本**
   - 如果用户提供了文件路径，读取文件
   - 如果用户直接粘贴了文本，使用粘贴内容
   - 如果都没有，列出 `00_收件箱/` 中的条目让用户选择

2. **识别内容类型和结构**
   - 判断来源类型：课堂笔记 / 读书摘录 / 会议记录 / 网页剪藏 / 自由笔记
   - 识别核心主题和领域
   - 提取关键概念（适合做原子概念卡片的）
   - 提取文献引用（适合放入 30_研究/ 的）
   - 提取行动项和想法（适合放入收件箱的）
   - 检查 vault 中是否已有相关笔记，避免重复

### Step 2: 展示解析方案（等待确认）

向用户展示计划：

```
## 解析方案

**原始内容:** [来源描述]
**核心主题:** [主题]

**将创建：**

1. 📄 主笔记: `30_研究/[主题].md`
   - [内容概要]

2. 🏷️ 概念卡片 (N 个):
   - [[概念A]] — 一句话说明
   - [[概念B]] — 一句话说明

3. 📝 收件箱条目 (N 个):
   - [待跟进的想法或行动项]

**将更新：**
- [[已有概念C]] — 补充新内容

确认后开始创建，或告诉我要调整什么。
```

### Step 3: 执行创建（用户确认后）

#### 主笔记
- 路径：根据内容类型决定
  - 文献相关 → `30_研究/`
  - 主题研究 → `30_研究/`
  - 项目相关 → 链接到对应项目
- 重写原始文本使其结构化、模块化
- 核心概念用 `[[wikilink]]` 链接到概念卡片

#### 概念卡片
- 路径：`40_概念/[概念名].md`
- frontmatter:
  ```yaml
  ---
  type: concept
  created: YYYY-MM-DD
  related: [[相关概念]]
  source: parse-knowledge
  ---
  ```
- 内容：简洁的概念定义（1-3 段），包含定义、背景、与其他概念的关系
- 在已有概念卡片中标注相关链接

#### 收件箱条目
- 文本中提到的待跟进想法 → `00_收件箱/`
- 文本中提到的行动项 → 今日日记待办

### Step 4: 处理原文件
- 如果来自收件箱，更新 `status: processed`
- 保留原文件不删除

### Step 5: 汇报

```
## 知识解析完成

**主笔记:** [[主题]] 位于 30_研究/

**新建概念卡片 (N):**
- [[概念A]] — 一句话说明
- [[概念B]] — 一句话说明

**更新概念卡片 (N):**
- [[已有概念C]] — 补充了新内容

**关联关系:**
- 主笔记链接到 N 个概念
- 建立了 M 个概念间的交叉引用

**收件箱新增 (N):**
- [[待跟进想法]]

**推荐下一步:**
- /explore [[概念A]] — 深入了解这个概念
- /read — 精读相关文献
```

## 注意
- **保持原始信息**：重构结构但不丢失信息
- **概念卡片要原子化**：一个卡片只讲一个概念
- **大量使用 wikilinks**：概念之间、笔记之间要充分链接
- **不要过度拆分**：3-5 个概念卡片通常就够了，不要碎片化
- **先展示方案再执行**：让用户确认后再创建文件
- **检查已有笔记**：避免创建重复的概念卡片
